Implementing the IP Module Using ispLEVER


Instantiating the Core

The generated SGDMAC IP core package includes black-box (<user_name>_bb.v) and instance (<username>_inst.v) templates that can be used to instantiate the core in a top-level design.

An example RTL top-level reference source file that can be used as an instantiation template for the IP core is provided in C:\<project_dir>\sgdmac_eval\<username>\src. Users may also use this top-level reference as the starting template for the top-level for their complete design.

To instantiate this IP core on a Linux/Unix platform, the user should manually add one environment variable named "SYNPLIFY" to indicate the installation path of the SYNPLIFY TOOLS in the local environment file.

For example: setenv SYNPLIFY =/tools/local/ispTOOLS8_0/isptools/synplify_linux


Hardware Evaluation

Lattice's IP hardware evaluation capability makes it possible to create IP cores that operate in hardware for a limited period of time (approximately four hours) without requiring the purchase on an IP license. The hardware evaluation capability is enabled by default. It can be disabled by right clicking on "Build Database" in the "Process for current sources" window of the Project Navigator. The setting is called "Hardware Evaluation" and the options are "Enable" or "Disable".
 
When the Hardware Evaluation feature is enabled in the design, it will generate a programming file that may be downloaded into the device. After initialization, the IP core will be operational for approximately four hours. After four hours, the device will stop working and it will be necessary to reprogram the device to re-enable operation. This hardware evaluation capability is only enabled if the core has not been licensed. During implementation, a license check is performed. If the hardware evaluation feature is disabled, a pop-up window will be displayed indicating a license failure. Click"OK" in the window and the bitstream will not be generated. If a license is detected, no pop-up window is displayed and core generation is completed with no restrictions.


Implementing the Core in a Top-Level Design

As described previously, the top-level file sgdmac_top.v/vhd provided in C:\<project_dir>\sgdmac_eval\<username>\src supports the ability to implement just the Scatter Gather DMA Controller IP core.

Push-button top-level implementation of this top-level is supported via the ispLEVER project file <username>_eval.syn located in C:\<project_dir>\sgdmac_eval\<username>\impl.

This design is intended only to provide an accurate indication of the device utilization associated with the core itself and should not be used as an actual implementation example.


For the Linux/Unix platform, the top-level synthesis needs to be run separately with Synplify Pro, as the ispLever Linux/Unix version only accepts an EDIF entry. Synthesis tcl files are provided for this purpose including sgdmac_top.tcl in the \<project_dir>\sgdmac_eval\<username>\impl directory.

In order to generate edif on the Linux/Unix platform from a GUI interface, the user should do the following:

  • Type “$SYNPLIFY/bin/synpwrap –gui” for OEM synplify pro or  “$SYNPLIFY/bin/synpwrap –gui -notPro” for OEM synplify.
  • Select “Run” and “Run Tcl scripts”
  • Browse to \<project_dir>\sgdmac_eval\<username>\impl
  • Choose sgdmac_top.tcl and click Open

 

In order to generate edif on the Linux/Unix platform from a command line, the user should do the following:

  • Change to the \<project_dir>\sgdmac_eval\<username>\impl directory
  • Type “synplify_pro –batch sgdmac_top.tcl”

 

To use the project file:

  • Select Open Project under the File tab in ispLEVER.
  • Browse to \<project_dir>\sgdmac_eval\<username>\impl in the Open Project dialog box.
  • Select and open <username>_eval.syn. At this point, all of the files needed to support top-level synthesis and implementation will be imported to the project.
  • Implement the complete design via the standard ispLEVER GUI flow.

 


Running Functional Simulation

The functional simulation includes a configuration-specific behavioral model of the SGDMAC IP Core. This FPGA top is instantiated in an eval testbench that configures FPGA test logic registers and SGDMAC IP core registers.


The generated IP core package includes the configuration-specific behavior model (<username>_beh.v) for functional simulation. ModelSim simulation is supported via testbench files provided in C:\<project_dir>\sgdmac_eval\sim\modelsim\testbench. Models required for simulation are provided in the corresponding \models folder.


Users may run the eval simulation by doing the following with ModelSim:

  • Open ModelSim.
  • Under the File tab, select Change Directory
  • Set the directory to \<project_dir>\sgdmac_eval\sim\modelsim.
  • Select OK.
  • Under the Tools tab, select Execute Macro
  • Select file sgdmac_eval.do

 

Users may run the eval simulation by doing the following with Active-HDL:

  • Open Active-HDL.
  • Under the tools tab, select Execute Macro...
  • Select file \<project_dir>\sgdmac_eval\sim\aldec\sgdmac_eval.do
